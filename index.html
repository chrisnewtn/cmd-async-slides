Regular Approach

Markup script elements one after the other.

Scripts are loaded in sequence.

Scripts coupled with HTML.

Limited to no flow control.

Typical HTML page

<html>
  <head>
    <!-- omitted -->
    <script src="jquery.js"></script>
    <script src="jquery-plugin-1.js"></script>
    <script src="jquery-plugin-2.js"></script>
    <script src="underscore.js"></script>
    <script src="site-script.js"></script>
    <!--[if lt IE 9]><script src="shiv.js"></script><![endif]-->
  </head>
  <body>
    <!-- omitted -->
    <script src="google-analytics.js"></script>
    <script src="twitter-widget.js"></script>
    <script src="flickr-widget.js"></script>
  </body>
</html>

Why use cmd.js?

Bit more succinct...

<html>
  <head>
    <!-- omitted -->
  </head>
  <body>
    <!-- omitted -->
    <script src="cmd.js"></script>
    <script src="site-script.js"></script>
  </body>
</html>

Examples

Load a single script

cmd('im-important.js');

Load & execute scripts one after the other

cmd('he-needs-me.js', 'i-need-him.js');

Load scripts in parallel

cmd(['i-dont-need-nothing.js', 'i-will-fire-when-ready.js']);
So far so good...

Callbacks

Just one callback.

cmd('prettify.js', function () {
  prettyPrint();
});

Many callbacks!

cmd(
  'does-nothing-onload.js',
  nowItDoes,
  'also-does-nothing.js',
  nowItDoesToo
);
